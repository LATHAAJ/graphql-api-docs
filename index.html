About GraphQL

The GraphQL data query language is:

  • A specification. The spec determines the validity of the schema on the API server. The schema determines the validity of client calls.
  • Strongly typed. Every GraphQL service defines a complete type system for the data it serves.
  • Introspective. A GraphQL service can be queried for the types it supports.

Getting Started

This GraphQL API provides comprehensive management for cricket players, teams, matches, and statistics. The API supports:

  • Player Management: Create, read, update, and delete player information
  • Team Operations: Manage team affiliations and player transfers
  • Match Tracking: Record and track cricket matches
  • Statistics: Detailed player performance analytics
💡 Tip: Use the interactive documentation to explore the API schema and test queries.

Available Queries

Field Type Description
findAll [Player] Retrieve all cricket players in the system
findPlayerById Player Look up a specific player by their unique identifier
findPlayersByTeam [Player] Get all players belonging to a specific team
findPlayersByRole [Player] Find players by their role (Batsman, Bowler, etc.)
findAllMatches [Match] Retrieve all cricket matches
findMatchById Match Look up a specific match by its unique identifier
findPlayerStats PlayerStats Get detailed statistics for a specific player

Available Mutations

Field Type Description
createPlayer Player Create a new cricket player
updatePlayer Player Update existing player information
deletePlayer Boolean Delete a player from the system
createMatch Match Create a new cricket match
updatePlayerStats PlayerStats Update player performance statistics

Core Types

Player

Represents a cricket player with their basic information and team affiliation.

Match

Represents a cricket match with teams, date, and result information.

PlayerStats

Contains detailed performance statistics for a player.

Authentication

This API uses standard HTTP authentication. Include your API key in the request headers:

Authorization: Bearer YOUR_API_KEY

Rate Limits

The API has the following rate limits:

  • 1000 requests per hour for authenticated users
  • 100 requests per hour for unauthenticated users
  • Rate limit headers are included in all responses
